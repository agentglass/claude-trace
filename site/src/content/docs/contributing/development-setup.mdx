This guide assumes zero prior knowledge of the codebase. Follow every step in order.

## Prerequisites

Before you begin, you need:

- **Python 3.11 or 3.12** — claude-trace requires 3.11+. Check with `python3 --version`.
- **pip** — comes with Python. Check with `pip --version`.
- **git** — for cloning and branching. Check with `git --version`.

Optional but recommended:
- **pyenv** — for managing multiple Python versions: https://github.com/pyenv/pyenv
- **VS Code** with the Python extension — for IDE support

## Step-by-step setup

<Steps>

1. **Fork and clone the repository**

   Fork `https://github.com/claude-trace/claude-trace` on GitHub, then clone your fork:

   ```bash
   git clone https://github.com/YOUR_USERNAME/claude-trace.git
   cd claude-trace
   ```

2. **Create a virtual environment**

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate   # on Windows: .venv\Scripts\activate
   ```

   Verify you're in the venv:
   ```bash
   which python   # should show .venv/bin/python
   python --version   # should be 3.11+
   ```

3. **Install the package in editable mode with dev dependencies**

   ```bash
   pip install -e ".[dev]"
   ```

   This installs:
   - The `claude_trace` package itself (editable, so code changes take effect immediately)
   - `anthropic>=0.40.0`
   - `opentelemetry-api>=1.25.0` and `opentelemetry-sdk>=1.25.0`
   - `pytest>=8.0` and `pytest-asyncio>=0.23`
   - `mypy>=1.9` (strict type checker)
   - `ruff>=0.4` (linter + formatter)
   - `opentelemetry-exporter-otlp>=1.25.0`
   - `rich>=13.0`

4. **Verify the installation**

   ```bash
   python -c "import claude_trace; print(claude_trace.__version__)"
   ```

   Expected output: `0.1.0` (or the current version).

5. **Run the test suite**

   ```bash
   pytest
   ```

   Expected output:
   ```
   ========================= test session starts ==========================
   collected 18 items

   tests/unit/test_semconv.py ..........                            [ 55%]
   tests/unit/test_cost_calculator.py ........                      [100%]

   ========================= 18 passed in 0.42s ===========================
   ```

   All 18 tests should pass. If any fail, the most likely cause is a Python version incompatibility or a missing dependency. Check `pip list` to confirm all packages installed.

6. **Run the type checker**

   ```bash
   mypy src/
   ```

   Expected output: `Success: no issues found in N source files`

   If you see errors, they indicate a type issue in the current codebase (please report it) or a mypy version incompatibility.

7. **Run the linter**

   ```bash
   ruff check src/ tests/
   ```

   Expected output: nothing (no errors). Ruff is fast — this should complete in under 1 second.

8. **Set up your ANTHROPIC_API_KEY (for integration testing)**

   For tests that make real API calls (there are none in the unit test suite, but you may want to add them):

   ```bash
   export ANTHROPIC_API_KEY="sk-ant-..."
   ```

   Or add it to a `.env` file (not committed to git):
   ```
   ANTHROPIC_API_KEY=sk-ant-...
   ```

9. **Build the documentation site (optional)**

   ```bash
   cd site
   npm install
   npm run dev
   ```

   Open http://localhost:4321 to see the docs site locally.

</Steps>

## Project structure walkthrough

Once set up, here is what each file does:

```
claude-trace/
├── pyproject.toml              # Build config, dependencies, tool settings
├── src/claude_trace/           # The library (everything users import)
│   ├── __init__.py             # Public API surface (4 functions)
│   ├── config.py               # TraceConfig - all configuration
│   ├── context.py              # contextvars session propagation
│   ├── _semconv/claude.py      # All ~50 attribute name constants
│   ├── _spans/                 # Span creation for each level
│   │   ├── session.py          # Root span + cumulative stats
│   │   ├── turn.py             # Per-call span + token recording
│   │   ├── tool.py             # Per-tool span + hash/sizing
│   │   └── attributes.py       # Re-exports for convenient import
│   ├── _instrumentation/       # The monkey-patching layer
│   │   ├── __init__.py         # instrument()/uninstrument()
│   │   ├── sync_client.py      # Patches Messages.create
│   │   └── async_client.py     # Patches AsyncMessages.create
│   ├── _cost/                  # Token cost calculation
│   │   └── calculator.py       # 16-model pricing table + CostBreakdown
│   └── _diff/                  # Trace comparison
│       └── trace_diff.py       # TraceSnapshot + TraceDiff + compare()
└── tests/
    └── unit/
        ├── test_semconv.py         # Attribute name format tests
        └── test_cost_calculator.py # Pricing math tests
```

## VS Code setup

For the best IDE experience:

1. Install the **Python** extension (ms-python.python)
2. Install the **Pylance** extension (ms-python.vscode-pylance)
3. Install the **Ruff** extension (charliermarsh.ruff)

Add to your `.vscode/settings.json`:

```json
{
    "python.defaultInterpreterPath": "${workspaceFolder}/.venv/bin/python",
    "python.analysis.typeCheckingMode": "strict",
    "[python]": {
        "editor.defaultFormatter": "charliermarsh.ruff",
        "editor.formatOnSave": true,
        "editor.codeActionsOnSave": {
            "source.fixAll.ruff": "explicit",
            "source.organizeImports.ruff": "explicit"
        }
    }
}
```

## Common setup problems

### `ModuleNotFoundError: No module named 'anthropic'`

You're not in the virtual environment, or the install didn't complete. Run:
```bash
source .venv/bin/activate
pip install -e ".[dev]"
```

### `pytest: command not found`

pytest is not installed in the active environment. Run `pip install pytest pytest-asyncio`.

### `mypy` reports `Cannot find implementation or library stub for module named 'anthropic'`

Install the anthropic stubs: `pip install anthropic`. The `anthropic` package bundles its own type stubs.

### Python 3.10 compatibility errors

claude-trace uses `match` statements (Python 3.10+) and `from __future__ import annotations`. If you see `SyntaxError` on `match`, you're running Python 3.9 or older. Upgrade with pyenv:

```bash
pyenv install 3.12.0
pyenv local 3.12.0
python3 -m venv .venv
```
