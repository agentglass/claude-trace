When you install the claude-trace plugin into a Claude Code project, every tool Claude Code executes
is **automatically traced as an OTel span** — with no changes to your code and no decorators to add.

This page explains how the hook system works and how to configure it.

## How It Works

Claude Code fires **PreToolUse** and **PostToolUse** hook events for every tool invocation
(Write, Read, Bash, Edit, Glob, Grep, …). The claude-trace plugin registers lightweight Python
commands that turn each event pair into a completed `claude.tool.invocation` OpenTelemetry span.

## Span Reference

Every tool invocation produces a `claude.tool.invocation` span with these attributes:

<div class="attr-table">
  <div class="attr-row attr-row--header">
    <span>Attribute</span><span>Type</span><span>Description</span>
  </div>
  <div class="attr-row">
    <code>claude.tool.name</code><span>string</span><span>Tool name (Write, Bash, Read, …)</span>
  </div>
  <div class="attr-row">
    <code>claude.tool.input_hash</code><span>string</span><span>SHA-256[:16] of tool input — never the raw input</span>
  </div>
  <div class="attr-row">
    <code>claude.tool.output_hash</code><span>string</span><span>SHA-256[:16] of tool output — never the raw output</span>
  </div>
  <div class="attr-row">
    <code>claude.tool.duration_ms</code><span>float</span><span>Wall-clock execution time in milliseconds</span>
  </div>
  <div class="attr-row">
    <code>claude.tool.outcome</code><span>string</span><span>"success" or truncated error message (≤128 chars)</span>
  </div>
  <div class="attr-row">
    <code>claude.tool.session_id</code><span>string</span><span>Claude Code session identifier</span>
  </div>
  <div class="attr-row">
    <code>claude.tool.hook_source</code><span>string</span><span>Always "claude_code_plugin"</span>
  </div>
  <div class="attr-row">

  </div>
</div>

## Quick Setup

### 1. Install the plugin

```bash
# From the claude-trace repo root
claude plugin install .

# Or from the registry (when published)
claude plugin install claude-trace
```

### 2. Set your OTLP endpoint

```bash
# Jaeger all-in-one (local dev)
docker run -p 4318:4318 -p 16686:16686 jaegertracing/all-in-one:latest

export OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4318
```

### 3. Start a Claude Code session

Open any project with Claude Code. Every tool call is now traced automatically:

```
🔍 Open Jaeger at http://localhost:16686
   Service: claude-code
   Spans:   claude.tool.invocation
```

    # OTLP collector endpoint (default: http://localhost:4318)

    export CLAUDE_TRACE_HOOKS_LOG=/tmp/claude-tool-spans.ndjson

## Privacy & Security

<Aside type="caution" title="What is and isn't stored">
  claude-trace hooks **never** store the raw content of tool inputs or outputs.
  Only the SHA-256 hash (first 16 hex characters) is recorded. This lets you
  correlate invocations across runs without ever capturing file contents, shell
  commands, or code diffs.
</Aside>

| Data | Stored? |
|---|---|
| Tool name | ✅ Yes |
| Tool input content | ❌ Never — hash only |
| Tool output content | ❌ Never — hash only |
| Duration | ✅ Yes |
| Outcome (success/error) | ✅ Yes |
| Error message | ✅ Truncated to 128 chars, API keys redacted |
| Claude Code session ID | ✅ Yes |

### State File Security

State files written by the pre-hook use permission mode `0o600` (owner read/write only).
The state directory (`/tmp/claude-trace/`) is also created with `0o700`.
On Linux, `/tmp` is typically world-readable, so you can override the directory:

```bash
export CLAUDE_TRACE_STATE_DIR=/run/user/$UID/claude-trace
```

## Debugging

### No spans in Jaeger?

```bash
# 1. Check if hooks are firing
export CLAUDE_TRACE_HOOKS_LOG=/tmp/hook-debug.ndjson

# 2. Manually simulate a pre/post pair
echo '{"session_id":"test","tool_name":"Write","tool_input":{"f":"x"}}' \
  | python -m claude_trace.hooks pre

echo '{"session_id":"test","tool_name":"Write","tool_response":"ok"}' \
  | python -m claude_trace.hooks post

# 3. Check the fallback log
cat /tmp/hook-debug.ndjson | python -m json.tool
```

### Verify OTLP connectivity

```bash
curl -X POST http://localhost:4318/v1/traces \
  -H 'Content-Type: application/json' \
  -d '{"resourceSpans":[]}'
# Expected: 200 OK with {"partialSuccess":{}}
```

## Under the Hood

The hooks are implemented in `python/claude_trace/_hooks.py` (core logic) and
`python/claude_trace/_hooks_cli.py` (CLI entry point). The OTLP exporter uses
only Python standard library (`urllib.request`) — no extra dependencies.

State correlation between pre/post hooks uses the filesystem:
- **Pre-hook** writes `{state_dir}/{session_id}/{tool_name}.json`
- **Post-hook** reads, exports, then deletes that file

This is safe for sequential tool execution (Claude Code's default behaviour).
If you need concurrent tool support, open a [GitHub discussion](https://github.com/agentglass/claude-trace/discussions).
